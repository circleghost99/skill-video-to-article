# OpenClaw AI 分身系統：從碎片刻畫到可執行技能的四層蒸餾機制

在現代數位工作中，我們每天產生大量的對話與資訊碎。傳統的筆記系統往往只能「儲存」而不能「轉化」。本影片介紹了一套基於 OpenClaw 與 Obsidian 的 AI 分身系統，旨在解決資訊過載與知識內化的斷層問題。作者的核心動機是讓 AI 透過日常交互真正「懂你」，並將靜態的資料自動演化為可執行的行動能力（Skills），實現「越用越好用」的正向循環。

## 雙入口設計：打破對話框的輸入限制

**機制是什麼？**
系統提供兩個平行的輸入管道：標準的聊天介面與桌面端實體化的「河馬」掛件。

**具體細節為何？**
1. **聊天入口**：支援任何與 OpenClaw 的日常對話與任務執行。
2. **河馬入口（桌面寵物）**：這是一個極具創意且高效的 UI。使用者可以將 PDF、長文本、網址連結直接拖入或貼給這隻張著嘴巴的河馬。它像是一個本地化的「Dropbox」，但資料不流向雲端，而是進入個人系統。

**為什麼重要？**
真實工作流中，高價值的資訊不僅來自對話，更多隱藏在網頁和文件中。河馬入口將「收藏」與「消化」的動作無縫銜接，確保了原材料攝入的廣度與便利性。

## 四層蒸餾：從原材料到長期記憶的進化鏈

**機制是什麼？**
資訊進入系統後，並非原樣堆砌，而是經歷 Main Intake -> Formal Markdown -> Distillation Queue -> Long-term Memory 的漏斗式篩選。

**具體細節為何？**
*   **Main Intake**：第一站分流。系統會自動將輸入標註為 Task（任務）、Decision（決策）、Content Candidate（內容候選）或 Research Material（研究資料）。這一步是將「混亂」初步轉化為「結構」。
*   **Formal Markdown**：系統判斷具備沉澱價值的資訊會被掛載到更正式的頁面。
*   **Distillation Queue（蒸餾隊列）**：這是系統的精華。AI 會將長篇內容「再壓一壓」，提煉成更短、更適合長期檢索的候選記憶。
*   **審批四選項**：在最終進入長期記憶前，使用者（或系統自動）可進行四種操作：**同意、合併、升級、丟掉**。

**為什麼重要？**
這套機制模仿了人類大腦的記憶過程：並非所有訊息都值得記住。透過層層過濾，系統確保了長期記憶的高純度，避免了數位垃圾的堆積。

## 從記憶到技能：將知識轉化為自動化行動

**機制是什麼？**
記憶在長期系統中並非靜止，當某種處理模式被驗證有效且重複出現時，會被自動升級為 OpenClaw 的原生技能（Skills）。

**具體細節為何？**
在 Obsidian 的 `Main Skill Pack` 頁面中，可以看到已養成的技能。例如影片展示的 `content blueprint`（內容藍圖生成）。每個 Skill 都包含明確的可靠度說明、解決的問題、以及觸發條件。這些 Skill 不僅是文字紀錄，它們會被發布為 OpenClaw 可以直接調用的行動指令。

**為什麼重要？**
這是「AI 分身」的終極體現。知識不再只是被「檢索」，而是變成了「能力」。這讓 OpenClaw 從一個單純的對話工具變成了真正的個人效能槓桿。

## 客製化與環境邊界：打造專屬的分身

**機制是什麼？**
系統允許使用者透過五個核心維度來定義 AI 的行為偏好，而非硬性編碼。

**具體細節為何？**
客製化需要告訴系統：
1. 最希望處理的事務類型。
2. 希望長期留下的內容類型。
3. 應過濾/忽略的內容。
4. 常見的重複工作。
5. 未來希望擅長的方向。

**技術限制提醒**：目前該系統在 macOS 上支援完整功能，但在 Linux 環境下尚不支援「河馬」桌面入口。

**為什麼重要？**
這五個問題定義了 AI 的「人格」與「專業領域」。透過明確邊界與目標，使用者能確保 AI 的進化方向與個人成長路徑保持一致。

## 結論與洞察：筆記系統的範式轉移

這套系統最深刻的啟示在於：**未來的筆記系統不應該是死掉的檔案庫，而應該是活著的技能工廠。** 透過 Obsidian 作為透明的可視化界面，使用者保留了對知識的控制與修改權，而 OpenClaw 則在背後負責繁重的蒸餾與轉化工作。這種「人機協同蒸餾」的模式，為解決資訊焦慮提供了一條可執行的路徑。
